# CEO-Reviewed Plan: Web-First After-Session OS

## Summary

Собрать **web-first MVP** для демонстрации директору центра: индивидуальный специалист после занятия загружает или записывает голосовую заметку, система делает транскрипт, внутреннюю заметку, сообщение родителю и сохраняет результат в ограниченную обезличенную историю ребёнка.

CEO-позиционирование: это не “AI пишет заметки”, а **after-session OS** — маленькая система, которая закрывает самые болезненные минуты после занятия: фиксация, формулировка для родителя, история прогресса.

## Key Decisions

- Первый buyer для проверки: **директор центра**.
- Первый пользователь: **индивидуальный коррекционный специалист**, который работает с ребёнком один-на-один.
- Главная боль v1: **экономия времени специалиста после занятия**.
- MVP surface: **web-first demo**, не Telegram.
- Память: только **обезличенная история** по тестовому ребёнку, не полноценная база детей.
- AI-выводы: только **черновики**, финальный текст подтверждает человек.

## MVP Scope

- Web-интерфейс с карточкой обезличенного ребёнка.
- Загрузка аудио или ввод расшифровки вручную для демо.
- Транскрибация русской речи, если аудио включено в реализацию.
- Генерация:
  - внутренней заметки специалиста;
  - короткого сообщения родителю;
  - краткого обновления истории занятия.
- История из 3–5 занятий по обезличенному ребёнку.
- Простые вопросы к истории:
  - “Что улучшилось?”
  - “Что повторяется?”
  - “Что предложить родителю дома?”
- Редактирование и ручное подтверждение перед сохранением.

## Deferred

Не включать в первый MVP, но держать как сильные follow-up:

- Source-linked trust layer: пометки “из голоса”, “из истории”, “AI-предложение”.
- ROI-блок для директора: оценка сэкономленных часов и управленческой пользы.
- Пилотный пакет: сценарий демо, критерии успеха, safety рамка, предложение пилота.
- Telegram-first workflow.
- Полная CRM: реальные дети, роли, права доступа, панель директора, отчёты.

## Test Plan

- Проверить 3 обезличенных сценария:
  - обычный прогресс;
  - сложное занятие/регресс;
  - занятие с домашним заданием.
- Проверить, что AI не добавляет факты, которых нет во входных данных.
- Проверить, что сообщение родителю понятное, спокойное и без лишней терминологии.
- Проверить, что история отвечает только по сохранённым занятиям.
- Проверить empty states:
  - нет истории;
  - плохой/короткий ввод;
  - неполная заметка;
  - ошибка транскрибации или генерации.

## CEO Review Verdict

Текущий scope правильный: **не расширять до платформы центра** и не резать до одноразового генератора. Web-first MVP даст управляемое демо, а after-session OS framing сохранит амбицию продукта.

Критический риск: без наблюдения реального post-session workflow MVP всё ещё строится на частичном знании процесса. Перед первым серьёзным пилотом нужно посмотреть или подробно разобрать, как специалист реально пишет заметку и общается с родителем после занятия.
